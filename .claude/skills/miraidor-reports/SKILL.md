# ミライドア報告書作成スキル

面談メモ・文字起こし・箇条書きメモなどの入力から、ミライドア所定フォーマットの報告書を作成する。

## 報告書タイプの判別

入力内容から以下を自動判別する。判別できない場合のみユーザーに確認する。

- **初回面談報告書**: 新規のスタートアップとの面談。資金調達の相談、事業紹介、ピッチなどが主内容 → `templates/shokai-mendan.md` を使用
- **株主報告会まとめ**: 既存投資先の株主定例・株主報告会・決算報告。業績報告や予実の説明が主内容 → `templates/kabunushi-hokoku.md` を使用

## 共通ルール

1. **入力形式は不問**。メモ、文字起こし、箇条書きのいずれでも、テンプレートの該当項目に振り分ける
2. **情報がない項目は空欄にせず「−（未聴取）」と明記**する。これは次回面談での確認リストとして機能させるため
3. **数値は必ず原文から拾う**。定性的表現より定量表現を優先する（例:「順調に成長」ではなく「MoM+15%」）。原文に数字があるのに省略することは禁止
4. **事実と所感を分離**する。所感・評価はテンプレートの所定欄（総評等）のみに記載し、他のセクションには事実のみを書く
5. **簡潔な実務日本語**で書く。AI的な冗長表現を避ける。体言止め・箇条書きを活用する
6. **推測で埋めない**。原文にない情報を創作しない。不明確な発言は「（要確認）」を付す

## ワークフロー

1. 入力を受け取り、報告書タイプを判別する
2. 該当テンプレートを読み込む（templates/ 配下）
3. 入力内容をテンプレートに振り分けて報告書を作成し、まずチャット内にMarkdownで出力する
4. 未聴取項目が多い場合は、末尾に「次回確認事項リスト」として抜粋する
5. ユーザーの確認・修正を経て、希望があればWordファイル（Yu Gothic、A4縦、見出し階層付き）またはHTML（A4横向き印刷対応）として出力する

## 株主報告会に固有の処理

- **過去との比較が最重要**。過去の報告書・議事メモ・モニタリング資料が会話内やアップロードにあれば必ず参照し、「過去の議論からの変化・気づき」セクションを充実させる
- 過去情報が入手できない場合は、当該セクションに「（前回報告書の提供があれば比較可能）」と記載してユーザーに提供を促す
- **質疑応答は必ず記録**する。誰の質問か（当社／他株主）を区別し、回答が不十分だったものには「（回答不十分・継続確認）」を付す
- 決議事項がある場合は、当社の賛否と判断理由を必ず記載する
